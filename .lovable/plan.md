### Changes

**1. Rewrite the `.print-only` block in `src/pages/SavingsLedger.tsx`**

Replace the current compact print header (lines 195–219) with five distinct sections:

- **Brand bar** — left: `BrandLogo` (md, with name + tagline). Right: document title "SAVINGS PLAN STATEMENT" + generated date. Bottom border.
- **Holder strip** — Account Holder name (from `useAuth().profile.full_name`), email, and Plan ID short hash + reporting period (start → today). Two-column grid.
- **Plan Details** — Section heading + 2-column key/value grid (Plan Name, Type, Recipient, Frequency, Status, Started, Maturity, Note).
- **Financial Summary** — Section heading + a real `<table>` with 5 columns (Target, Saved, Remaining, Progress %, Next Due) so it renders as bordered cells via the global print table CSS. A small counts strip below: Paid · Pending · Overdue · Total.
- **Installment Schedule** — Section heading "INSTALLMENT SCHEDULE" placed above the existing on-screen installments table. Keep the existing table (already prints via global CSS); just add `print:table-header-group` so headers repeat across pages.
- **Footer** — `.print-only` footer line under the table: "Generated by MahBook · {date} · {appUrl}".

**2. Pull holder info from auth**
- Import `useAuth` from `@/contexts/AuthContext`. Use `profile?.full_name`, `profile?.email || user?.email`, `profile?.phone` (optional).

**3. Hide remaining on-screen chrome from print**
- The on-screen filters bar (`Select` for status) — already `no-print` ✓
- The "Generate next 12 installments" footer card — already `no-print` ✓
- Confirm the installments `<Card>` wrapper doesn't add unwanted padding in print → add `print:shadow-none print:border-none` to the wrapping `Card`.

**4. Tiny print CSS additions in `src/index.css`**
